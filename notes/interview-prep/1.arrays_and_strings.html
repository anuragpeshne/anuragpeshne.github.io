1 Hash Tables

1.1 Using arrays

  • Create hash, which is int or long using a hashing function.
    • since number of ints is less than the total possible keys, multiple keys will map to same integer
  • Calculate index into array using the hash modulo size of array.
    • Multiple hash will obviously clash to the same index
  • Handle collision using a linked list
  • On average case: O(1)
    • Worst Case: O(N)

1.2 Using Balanced Binary Search Tree

  • Search complexity: O(log₂ N)
  • Can iterate over entries in order.

3 Questions

3.1 IsUnique

Determine if a string has all unique characters. What if we cannot use additional data structures.

3.1.1 Ans1: Hash Table

We memorize the characters seens so far using a hash table. For each character in string, we mark the spot. If we encounter same character again, we return false.

  1. Python
    def all_unique_char(inp):
   memory = {}
   for c in inp:
       if c in memory:
           return False
       else:
           memory[c] = True

   # we are out of the loop, thus no repeated character found
   return True

inp = "test input string"
print(all_unique_char(inp))

    False
  2. C++
    #include <iostream>
#include <unordered_map>

bool all_unique_char(std::string& inp) {
    std::unordered_map<char, bool> memory;
    for (auto & c : inp) {
        if (memory.find(c) == memory.end()) {
            memory[c] = true;
        } else {
            return false;
        }
    }
    return true;
}

int main () {
    std::string inp1("test input string"), inp2("qwerty");
    std::cout << all_unique_char(inp1) << std::endl;
    std::cout << all_unique_char(inp2) << std::endl;
    return 0;
}

    0
1

    Notes:

    • Here c has type char. So for (auto & c : inp) is same as for (char & c : inp).
    • The loop can be written in a traditional way: for (int i = 0; i < memory.length(); i++), then access element memory[i].

3.1.2 Ans2: ASCII array

3.1.3 Introspection

To determine if string has unique character, we have to know for given character, have we seen the character before. For that either we can:

  1. remember the characters seen so far in a datastructure in which we can check fast, or the situation is degraded to choice 2.
    • Hash table is useful to map or remember arbitrary range of keys. But if the domain is well defined, we can map the keys to integers and use a array.
      • an array is, after all, a map, which maps natural number to values.
  2. if we do not remember, we'll have to check each character to characters seen so far by traversing over the input list. This is the case even if we remember using a linear search complexity data structure.
  3. Intuitively lower bound is O(n)

We can generalize the complexity as:
O(n × (lookup time))

∴ for hash table implemented using arrays and ASCII arrays:
O(n)

For tree based hash tables:
O(n × log₂ n)

For list with linear search:
O(n × n) = O(n²)

3.1.4 Ans3: Without additional data structure

  1. Python
    def all_unique_char(inp):
    for i in range(len(inp)):
        for j in range(i):
            if inp[i] == inp[j]:
                return False
    return True

print("Test1", all_unique_char("this is test"))
print("Test2", all_unique_char("al string"))

    ('Test1', False)
('Test2', True)
  2. C++
    #include <iostream>

bool all_unique_char(std::string &inp) {
    for (auto i = 0; i < inp.length(); i++) {
        for (auto j = 0; j < i; j++) {
            if (inp[i] == inp[j])
                return false;
        }
    }
    return true;
}

int main() {

    return 0;
}

3.2 IsPermutation

Given two strings, determine if one is a permutation of the other.

3.2.1 Understanding the problem

  • Given a string 'abc', its permutations are {'abc', 'acb', 'bac', 'bca'…}.
  • The characters in each permutation are same, position is irrelevant.

3.2.2 Devise a plan

  • ⇒ we need to remember characters in string 'a' and see if same characters appear in string 'b'.
    • We need to keep count of characters, to handle multiple occurrence of same character
  • Use an ASCII array to keep count of characters in string A and then decrement the count when character in b is encountered. If no count goes negative (no character in string b which is not present in string a) and all counters become zero at the end (all characters in string a are matched up with characters in string b) then they have same characters.
    • Compare length of a and b, return false if length mis match. This will help to eliminate the need to check if all counters are 0 at the end. If length is same, and suppose there is one character mismatch, then there will be one instance where counter will go negative.

3.2.3 Carry out the plan

  1. Python
    def is_permutation(str1, str2):
    if len(str1) != len(str2):
        return False

    mem = [0] * 256
    for c in str1:
        mem[ord(c)] = mem[ord(c)] + 1

    for c in str2:
        old_val = mem[ord(c)]
        if old_val < 1: # there should be at least one instance in a
            return False
        else:
            mem[ord(c)] = old_val - 1
    return True

print(is_permutation("abc", "bca"))
print(is_permutation("abc", "abv"))

    True
False
  2. C++

10:     }

14:             return false;

3.3 URLify

Given a string and its real length, replace all the spaces with '%20'. The input string is padded with enough spaces to accommodate the expanded string. Example: ('I am a teapot ', 13) → 'I%20am%20a%20teapot'.

3.3.1 Understanding the problem

  • We have to replace space with '%20'. To do this in place, we'll have to shift next characters.
  • Doing this using extra space (O(n)) is trivial (similar to fizz buzz).

3.3.2 Devise a plan

  • Using extra space (O(n)), it's trivial: if c == ' ': append('%20') else: append(c)
  • Let's try to solve it using constant space buffer:
    • buffer of size 2: queue.
    • when we encounter ' ', we enqueue next characters in queue
    • This won't work: if we encounter another ' ' before we empty the buffer, we'll need more space. If ' ' = O(n), then we'll need space = O(n)
  • If we start from right end, we need to shift characters to right which never over write other characters since enough space is appended in given string.

3.3.3 Carry out the plan

  1. Python
    def urlify(inp):
    result = []
    runner = len(inp) - 1
    while inp[runner] == ' ':
        runner = runner - 1

    worker = len(inp) - 1
    while runner >= 0:
        if inp[runner] == ' ':
            result.insert(0, '0')
            worker = worker - 1
            result.insert(0, '2')
            worker = worker - 1
            result.insert(0, '%')
            worker = worker - 1
        else:
            result.insert(0, inp[runner])
            worker = worker - 1
        runner = runner - 1
    return ''.join(result)

print("test1", urlify("I am a teapot      "))
print("test2", urlify("nospace"))

    ('test1', 'I%20am%20a%20teapot')
('test2', 'nospace')
  2. C++
    #include <iostream>

void urlify(std::string &inp) {
    int runner = inp.length() - 1;
    int worker = inp.length() - 1;

    while (inp[runner] == ' ') runner--;

    while (runner >= 0) {
        if (inp[runner] == ' ') {
            inp[worker--] = '0';
            inp[worker--] = '2';
            inp[worker--] = '%';
            runner--;
        } else {
            inp[worker--] = inp[runner--];
        }
    }
}

int main() {
    std::string inp1("I am a teapot      "), inp2("nospace");
    urlify(inp1);
    urlify(inp2);

    std::cout << inp1 << std::endl;
    std::cout << inp2 << std::endl;
}

    I%20am%20a%20teapot
nospace

3.3.4 Looking Back

  • Looping from right end proved to be easy in case of shifting characters.

3.4 Palindrome Permutation

Given a string, write a function to check if it is a permutation of a palindrome. 

Ex: "anastasta" → true
    (aastntsaa)

3.4.1 Understanding the problem

  • What is permutation: rearrangement of letters
    • order is unimportant
  • What is Palindrome: a string which is same forward and backward
    • each character, except one in the middle, should appear even number of times.

3.4.2 Devising a plan

  • The obvious way of generating all permutation (O(n!)) and checking if it is palindrome (O(n)) is very expensive.
  • In previous problem, 3.2, we observed that in a permutation, both strings have same characters and order is not important
    • Palindrome string imposes a new condition that all characters except one in the middle should appear even number of times.
  • Count the number of times each character appear, all, except one, should appear even number of times.

3.4.3 Execute the Plan

  1. Python
    def is_palindrome_permute(inp):
    store = [0] * (26 * 2)
    for c in inp:
        if ord(c) >= ord('A') and ord(c) <= ord('Z'):
            store[ord(c) - ord('A')] = store[ord(c) - ord('A')] + 1
        elif ord(c) >= ord('a') and ord(c) <= ord('z'):
            store[ord(c) - ord('a') + 26] = store[ord(c) - ord('a') + 26] + 1

    found_middle_char = False
    for i in store:
        if (i % 2 == 1):
            if found_middle_char:
                return False
            else:
                found_middle_char = True
    return True

print("anastasta", is_palindrome_permute("anastasta"))
print("notpalim", is_palindrome_permute("notpalim"))

    ('anastasta', True)
('notpalim', False)
  2. C++
    #include <iostream>

bool is_palindrome_permute(std::string &inp) {
    int store[(26*2)] = {0};

    for (auto c : inp) {
        if (c >= 'a' && c <= 'z') {
            store[c - 'a']++;
        } else if(c >= 'A' && c <= 'Z') {
            store[c - 'A' + 26]++;
        }
    }

    bool found_middle_char = false;
    for (auto i : store) {
        if (i % 2 == 1) { //odd number
            if (found_middle_char) {
                return false;
            } else {
                found_middle_char = true;
            }
        }
    }
    return true;
}

int main() {
    std::string inp1("anast asta"), inp2("notpalim");
    std::cout << inp1 << ":" << is_palindrome_permute(inp1) << std::endl;
    std::cout << inp2 << ":" << is_palindrome_permute(inp2) << std::endl;

    return 0;
}

    anast asta:1
notpalim:0

3.5 One Away

There are three types of edits that can be performed on text: insert, remove or replace. Given two strings write a function to check if they are one edit or less away. 

(app, appl) → true
(appl, app) → true
(appl, appr) → true

3.5.1 Understanding the Problem

  • Determine if two strings are 1 edit apart. Or 0 edits ie same.
  • Given two string either they are same or just by inserting, deleting or replacing one character we can make them equal.
  • Since we are interested in just one edit apart, then after fixing one edit, if we encounter another mismatch we can return false.
  • By insert or delete the length changes at most by one, replace keeps the length same.

3.5.2 Devising a plan

  • It is difficult to come up with an algorithm if we take small example:
    • since we can see the answer, coming up with an algorithm is difficult.
  • Since we are interested in knowing if strings are one or less edit apart, after fixing one mismatch if we face another, then we can return false
    • be careful of the options: insert, delete and replace.
      • insert and delete can shift the characters and make subsequent characters match.
      • we can try all 3 options to see if any one works.

  • Pseudocode 

    smaller, larger = magic(str1, str2)
if len(longer) - len(smaller) > 1: return False

for i in range(len(smaller)):
    if smaller[i] != longer[i]:
        return ((insert unmatched into smaller == longer) or
                (delete from longer == smaller) or
                (replace smaller[i] with longer == longer))
if len(shorter) != longer:
    shorter + longer[-1] == longer
else:
    return True

3.5.3 Carry out the plan

  1. Python
    def is_one_away(str1, str2):
    if len(str1) > len(str2):
        shorter, longer = str2, str1
    else:
        shorter, longer = str1, str2

    if len(longer) - len(shorter) > 1:
        return False

    for i in range(len(shorter)):
        if shorter[i] != longer[i]:
            if longer[i] + shorter[i + 1:] == longer: # this covers both insert and delete
                return True
            else:
                return shorter[i + 1:] == longer[i + 1:] # replace

    return True #no mismatch and either same len or off by 1

print("app", "appl", is_one_away("app", "appl"))
print("pp", "app", is_one_away("pp", "ppl"))
print("app", "apr", is_one_away("app", "apr"))
print("papp", "appla", is_one_away("papp", "appla"))
print("app", "aprl", is_one_away("app", "aprl"))

    ('app', 'appl', True)
('pp', 'app', True)
('app', 'apr', True)
('papp', 'appla', False)
('app', 'aprl', False)
  2. C++
    #include <iostream>

bool is_one_away(std::string &str1, std::string &str2) {
    std::string &shorter = (str1.length() < str2.length()) ? str1 : str2;
    std::string &longer = (str1.length() > str2.length()) ? str1 : str2;

    for (auto i = 0; i < shorter.length(); i++) {
        if (shorter[i] != longer[i]) {
            if (shorter.substr(i) == longer.substr(i + 1)) {
                return true;
            } else {
                return (shorter.substr(i + 1) == longer.substr(i + 1));
            }
        }
    }

    return true; // shorter is substring of longer and len is same
}

int main() {
    std::string t1i1("app"), t1i2("appl");
    std::string t2i1("app"), t2i2("aopp");
    std::string t3i1("app"), t3i2("apr");
    std::string t4i1("app"), t4i2("aprl");
    std::string t5i1("apr"), t5i2("appl");

    std::cout<< t1i1 << " " << t1i2 << ":" << is_one_away(t1i1, t1i2) << std::endl;
    std::cout<< t2i1 << " " << t2i2 << ":" << is_one_away(t2i1, t2i2) << std::endl;
    std::cout<< t3i1 << " " << t3i2 << ":" << is_one_away(t3i1, t3i2) << std::endl;
    std::cout<< t4i1 << " " << t4i2 << ":" << is_one_away(t4i1, t4i2) << std::endl;
    std::cout<< t5i1 << " " << t5i2 << ":" << is_one_away(t5i1, t5i2) << std::endl;
    return 0;
}

    app appl:1
app aopp:1
app apr:1
app aprl:0
apr appl:0

3.5.4 Looking Back

  • Insert and delete operation turned out to be same:
    • remove a character from longer str is same as adding character to shorter one.
  • If we were interested in know arbitrary edits or edit distance, then we may had to consider all 3 operations at each comparison in this algorithm. In fact each character would have to be compared with each character from other string.
  • You can assign C++ reference conditionally using ternary operator
  • C++ substr has similar interface to python [] operator.

3.6 String Compression

Implement a string compression using counts of repeated characters. Assume string has only uppercase and lowercase letters. 

aabcddd → a2bcd3

3.6.1 Understanding the problem

Count number of contiguous characters, append character with the count and print. If count is 1, no need to print the count.

3.6.2 Devising a plan

  1. Remember the current character, increment the count if same character encountered, output the count when character mismatches. Print the count only if count > 1.
    • Time: O(n)
    • Space: constant
  2. In first iteration count all the characters in a ASCII array, in second iteration dump the count from array.
    • Time: O(n)
    • Space: constant (26*2)
    • This won't work if there are multiple occurrence of same characters (Eg: 'aabaa')

3.6.3 Carry out the plan

  1. Python
    def compress(inp):
    ans = []

    last_char = inp[0]
    count = 0
    for c in inp:
        if c == last_char:
            count = count + 1
        else:
            ans.append(last_char)
            if count > 1:
               ans.append(str(count))
            last_char = c
            count = 1

    ans.append(last_char)
    if count > 1:
        ans.append(str(count))
    return ''.join(ans)

print("aabcddd", compress("aabcddd"))
print("aabbbbbcddde", compress("aabbbbbcddde"))
print("abcd", compress("abcd"))

    ('aabcddd', 'a2bcd3')
('aabbbbbcddde', 'a2b5cd3e')
('abcd', 'abcd')
  2. C++
    #include <iostream>
#include <sstream>
#include <string>

std::string compress(std::string &inp) {
    char last_char = inp[0];
    int count = 0;
    std::stringstream ans;

    for (auto c : inp) {
        if (c == last_char) {
            count++;
        } else {
            ans << last_char;
            if (count > 1)
                ans << count;

            last_char = c;
            count = 1;
        }
    }
    ans << last_char;
    if (count > 1)
        ans << count;
    return ans.str();
}

int main() {
    std::string inp1("aabcdddaaa"), inp2("aabbbbbcddde"), inp3("abcd");

    std::cout << inp1 << ":" << compress(inp1) << std::endl;
    std::cout << inp2 << ":" << compress(inp2) << std::endl;
    std::cout << inp3 << ":" << compress(inp3) << std::endl;

    return 0;
}

    aabcdddaaa:a2bcd3a3
aabbbbbcddde:a2b5cd3e
abcd:abcd

3.6.4 Looking Back

  • The main point here was to print character when the character under observation changes and in the end print the last character
  • Counting all the character would not work because of multiple occurrence or same character would mix up the count
  • Include <sstream> to use stringstream.

3.7 Rotate Matrix

Given an image represented by an N×N matrix, rotate the image by 90. Each pixel is 4 bytes. Can it be done in place.

3.7.1 Understanding the problem

To rotate the image, we have to shift the pixel (not transpose) clockwise/anticlockwise.

1 2 → 3 1
3 4   4 2
1 2 3   7 4 1
4 5 6 → 8 5 2
7 8 9   9 6 3
1 2 3 4   13 9 5 1
5 6 7 8   14 10 6 2
9 10 11 12 → 15 11 7 3
13 14 15 16   16 12 8 4

3.7.2 Devising a plan

  • What if we don't have to do it in place (auxiliary space allowed):

    • we start with first row and fill up last column 

      for i in range(len(A)):
    for j in range(len(A[0])):
        B[len(A[0]) - i][j] = A[i][j]
  • In place:
    • If we start with 1:
      • 1 → 4 (0, 0 → 0, 3)
      • 4 → 16 (0, 3 → 3, 3)
      • 16 → 13 (3, 3 → 3, 0)
      • 13 → 1 (3, 0 → 0, 0)
      • Next round starts with offset 1
        • everyone moves one closer to center
        • total rounds = length / 2

void rotate(std::vector<std::vector<int> > &inp) {
    for (int i = 0; i < (inp.size() / 2) + 1; i++) {
        int top = i;
        int right = inp[0].size() - 1 - i;
        int bottom = inp.size() - 1 - i;
        int left = i;
        for (int j = i; j < inp[0].size() - 1 - i; j++) {
            // top right
            int temp = inp[top + j][right];
            inp[top + j][right] = inp[top][left + j];

            // bottom right
            int temp1 = inp[bottom][right - j];
            inp[bottom][right - j] = temp;
            temp = temp1;

            //bottom left
            temp1 = inp[bottom - j][left];
            inp[bottom - j][left] = temp;
            temp = temp1;

            // top left
            inp[top][left + j] = temp;
        }
    }
}

  • Rotating elements was about swapping elements of matrix.

3.8 Zero Matrix

if an element of matrix is 0, set entire row and column to 0.

3.8.1 Understanding the problem

1 2 0 10   0 0 0 0
4 5 6 11   4 5 0 11
7 8 0 12 → 0 0 0 0
11 12 13 56   11 12 0 56

3.8.2 Devising a plan

  • We can iterate over the array and as soon as we find '0', we'll set the whole row and column to '0'.
  • Problem with this approach is, it would be difficult to differentiate the original 0 from the ones we set.
  • we can set elements to '-1' and after reach the end, again iterate to convert -1 to 0.
    • Be careful to change to -1 only if it is not 0. (don't change 0 to -1)

3.9 String Rotation

Assume we have isSubstring, find where a string is rotation of another string.

3.9.1 Understanding the problem

  • Say the given strings are reboot and bootre.
  • Rotation of string is a permutation of string such that relative order (mod len) is preserved.

3.9.2 Devising a plan

  • Treating rotation as permutation is a good start.
  • Along with permutation, order is also important. So our definition of ration becomes, two strings with same characters in same relative order.
    • Thus just counting the characters won't solve the problem.
    • On other hand, if we check relative ordering of all characters we have essentially checked if all characters are present.
  • For checking relative ordering, we can start with first character of a string and find it in second string. Then once aligned we can start comparing. If we reach end of string, loop around.
  • Other hint given in question is about using isSubstring, this would what we discussed above but would not loop around.
    • → double the string and apply isSubstring.
